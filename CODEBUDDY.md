# CODEBUDDY.md — CodeBuddy 入口

> 本文件是 CodeBuddy 的项目入口。它只把你引导到知识库的核心协议，
> **不重复定义**权限、工作流、角色或格式。真正的规则在下面列出的文件里。

## 你正在进入一个个人知识库

这是一个基于 Markdown 的个人知识库框架，以主题（Topic）为核心维度组织。
任何 AI agent 在与它交互前，**必须先阅读并遵守 `SOUL.md`**。

## 首次交互：强制执行（不可跳过）

每次新会话开始、在回应用户任何请求之前，按顺序读取以下文件，并按 `SOUL.md`
「首次交互协议」回显自检报告：

1. `SOUL.md` — 行为契约：权限规则、交互协议、项目结构
2. `_schema.md` — 笔记格式规范、命名约定、必填字段
3. `SKILL.md` — 完整工作流定义
4. `topics/` 目录列表 — 获取当前所有主题

按任务追加读取：

- 涉及「多 Agent 讨论 / 多视角分析」→ 额外读 `AGENTS.md`
- 涉及跨平台运行、迁移、调用器适配 → 额外读 `PLATFORMS.md`

## 权限红线（详见 SOUL.md）

- ✅ 可自由执行：读取、列举、搜索、总结、分析
- ⚠️ 需用户确认：新增笔记、追加 `_discussions.md`、更新 `_overview.md`、
  新建主题、修改 `SKILL.md`/`AGENTS.md`/`PLATFORMS.md`
- 🚫 严格禁止：删除文件、改写已有笔记正文、重命名/移动文件、覆盖写入、
  未经授权修改 `SOUL.md`/`_schema.md`

**任何写入操作前必须先展示完整预览或 diff，等待用户明确确认。**

## CodeBuddy 运行定位

- 兼容级别：Level 2（文件代理模式），详见 `PLATFORMS.md` §5；
- 不同 CodeBuddy 环境的工具、权限和记忆能力可能不同，以项目 Markdown 协议为准；
- 若支持任务列表或子 Agent，可作为增强能力使用，但不作为硬前提；
- 多 Agent 讨论遵守 `AGENTS.md` + `SKILL.md` 工作流 G。
