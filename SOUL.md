# SOUL.md — 知识库行为契约

> 这是一个基于 Markdown 的个人知识库框架。任何 AI agent 在与此知识库交互前，**必须先阅读并遵守本文件**。
>
> 本文件是框架模板。首次使用请按需替换其中的占位说明（如仓库地址、个人偏好），但**权限规则与交互协议建议保持不变**——它们是这套框架安全性的核心。

## 这是什么

这是一个基于 Markdown 文件的个人知识库，用于持续记录主人的所见所闻、所思所想。内容来自书籍、播客、网帖、视频等多种来源，以**主题（Topic）**为核心维度组织。

知识库由一组协议文件驱动：AI 读取这些文件后，就知道该如何帮你查询、录入、讨论和沉淀知识，而无需依赖某个特定平台的私有能力。

## 目录结构速览

```
knowledge-base/
├── SOUL.md          ← 你正在读的文件（行为契约）
├── SKILL.md         ← 知识库管理的完整工作流定义
├── AGENTS.md        ← 多 Agent 讨论的角色池、主题映射与发言规范
├── PLATFORMS.md     ← 跨平台运行说明（WorkBuddy/Cursor/CodeBuddy/Codex/Claude Code 等）
├── _schema.md       ← 数据格式规范（字段定义、命名约定）
├── .gitignore       ← Git 忽略规则
├── _templates/      ← 笔记模板（7 个：book/podcast/web/video/article/general + topic-overview）
├── _vault/          ← 原始材料存储（PDF/视频/图片/音频）
│   └── _index.md    ← 材料链接映射表
├── tools/           ← 通用能力扩展（每个 .md 文件定义一个具体能力）
│   └── scripts/     ← tools 依赖的脚本
├── .config/         ← 配置文件（凭证、token 缓存等；不进版本库）
├── _changelog/      ← 统一操作日志（所有 agent 写操作后追加结构化记录）
└── topics/          ← 主题知识库
    └── {topic-name}/
        ├── _overview.md      ← 主题总览
        ├── _discussions.md   ← AI 讨论记录
        └── note-*.md         ← 来源笔记
```

> 可选扩展（不在核心框架内，可自行添加）：Web 前端、移动端 App 等。如添加，请遵循"项目隔离"原则——前端/App 只读知识库内容，写入严格限制在各自项目目录内。

## 权限规则

### ✅ 可自由执行（无需确认）

- 读取任意文件内容
- 列出主题列表、来源列表
- 搜索和查询知识库内容
- 回答关于知识库结构和内容的问题
- 总结、对比、分析知识库中的信息

### ⚠️ 需要用户明确确认后才能执行

- **新增**笔记文件（必须先展示完整预览）
- **追加**内容到 `_discussions.md`（必须先展示要追加的内容）
- **更新** `_overview.md` 中的个人理解部分
- **存入**原始材料到 `_vault/` 并更新 `_index.md`
- **新建**主题目录
- **修改** `SKILL.md`、`AGENTS.md` 或 `PLATFORMS.md` 中的工作流、角色定义、主题映射、平台适配规则（必须先展示修改方案或 diff）

### 🚫 严格禁止（即使用户口头说"随便改"也不要做）

- **删除**任何文件或文件内容
- **修改**已有笔记的正文内容（追加新段落除外）
- **重命名**或**移动**任何文件/文件夹
- **未经用户明确授权修改** `SOUL.md`（本文件）
- **修改** `_schema.md`（格式规范）
- **覆盖写入**任何已有文件（应使用追加或新建）

## 交互协议

1. **写操作前必须预览**：在执行任何写入操作前，先向用户展示完整的文件内容预览，等待用户明确说"确认"/"可以"/"写入"等肯定回复后再执行。
2. **主题匹配优先**：新增笔记时，先检查 `topics/` 下是否已有匹配的主题，建议归入已有主题；如需新建主题，需说明理由。
3. **格式遵循 _schema.md**：所有新建文件必须严格遵循 `_schema.md` 中定义的格式规范。
4. **来源溯源**：每条笔记必须标注来源信息（来源类型、标题、作者等），不可遗漏。
5. **讨论同步建议**：在与用户就某个主题讨论完毕后，主动询问是否要将讨论要点同步到对应主题的 `_discussions.md` 中。
6. **多 Agent 讨论协议**：当用户要求"进入讨论"、"多 Agent 讨论"、"多视角分析"等时，读取 `SKILL.md` 的工作流 G 与根目录 `AGENTS.md`。解释型 Agent 发言前，应读取/参考自身角色对应的知识库主题，寻找相关笔记或旧讨论；如未找到直接关联，必须明确说明。
7. **跨平台运行协议**：当用户询问平台依赖、迁移到 Cursor/CodeBuddy/Codex/Claude Code、跨平台运行、调用器适配等问题时，读取 `PLATFORMS.md`，按其中的兼容级别、启动方式和降级策略分析。
8. **操作日志记录**：任何写操作完成后，立即在 `_changelog/YYYY-MM.md` 追加结构化记录（详见「操作日志协议」章节）。此为已确认写操作的自动副作用，无需再次确认。

## 操作日志协议（_changelog/）

知识库的记忆分为**两层**，职责严格分开：

- **共享层 `_changelog/`**：跨平台、跨 Agent 的**结构化操作账本**，记录"做了什么"（增/删/改了哪个文件）。所有 agent 共用，方便主人和各 agent 在同一处查看全部操作历史。
- **私有层**：各平台自己的记忆（如某些平台的本地 memory），记录叙事过程与平台特定细节。

**规则**：

1. 任何 agent（WorkBuddy / Claude Code / Cursor / Codex / CodeBuddy 等）对知识库执行**写操作后**，必须在 `_changelog/YYYY-MM.md` 追加一条结构化记录（格式见 `_changelog/README.md`）。
2. 该日志追加是**已确认写操作的自动副作用**，本身**无需再次向用户确认**；纯读取/查询不记录。
3. **互不干预**：每个平台维护自己的私有 memory，agent 不得读写、修改或清理其他平台的私有 memory；跨平台唯一的共享渠道是 `_changelog/`。
4. 首次写操作前应读取 `_changelog/README.md` 确认格式与当月文件。

## 截图/文字录入协议

当用户发送截图或粘贴文字并希望录入知识库时：

1. **识别内容**：分析截图中的文字或粘贴的文本
2. **书籍截图增强**：如识别为书页截图，尝试在 `_vault/` 中查找电子版原文，结合原文上下文做总结（详见 SKILL.md 工作流 B）
3. **推断元数据**：判断来源类型、提取标题/作者等信息
4. **匹配主题**：检查已有主题，建议归属
5. **生成预览**：按模板生成完整笔记预览
6. **等待确认**：用户确认后才写入文件
7. **存档原图**：如果是截图，询问是否需要将原图存入 `_vault/images/`

## Tools — 通用能力扩展

`tools/` 目录下的每个 `.md` 文件定义一个独立的 AI 能力。使用规则：

- **按需加载**：根据当前任务读取对应的 tool 文件，不需要全部加载
- **自包含**：每个 tool 文件包含完整的触发条件、工作流和注意事项
- **权限不变**：所有 tool 的执行仍受本文件（SOUL.md）的权限规则约束
- **查看索引**：读取 `tools/README.md` 可获取当前所有可用 tool 的列表

`SKILL.md` 包含知识库管理的完整工作流定义（查询、截图录入、讨论同步、多 Agent 讨论与沉淀、每日知识库讨论、原始材料管理）。如需了解具体操作步骤，请阅读该文件。

## Git 版本同步

本知识库建议使用 Git 管理版本。Agent 在对知识库进行写操作后，需要遵循以下规则：

### 何时提交

在一次会话中，当**累计完成以下任意条件**时，应主动询问用户是否要提交并推送，用户没有答应则不能提交：

- 新增或修改了 **3 个及以上** 文件
- 新增了 **1 个主题目录**
- 完成了一轮完整的操作（如：批量录入笔记、讨论同步、工具配置修改等）
- 用户主动要求提交

### 如何提交

```bash
cd "知识库根目录"
git add .
git commit -m "简短描述本次变更"
git push
```

### commit message 规范

采用简短描述，格式：`类型: 内容`

| 类型 | 用途 | 示例 |
|------|------|------|
| `add` | 新增笔记/主题 | `add: 社会学主题下新增某书笔记` |
| `update` | 更新已有内容 | `update: 音乐史讨论记录同步` |
| `config` | 配置/工具变更 | `config: 更新工具代理设置` |
| `fix` | 修复问题 | `fix: 修复封面图路径` |
| `feat` | 新功能/新工具 | `feat: 新增下载工具` |

### 注意事项

- **隐私优先**：默认 `.gitignore` 已排除 `.config/`（凭证）等敏感目录。提交前请确认未把凭证或不想公开的内容纳入版本库。
- **不要修改 Git 配置**（如 user.name、user.email），除非用户明确要求。
- **大文件策略由用户决定**：`_vault/` 下的书籍/视频等大文件是否纳入 Git，取决于你的仓库容量与隐私偏好，请在 `.gitignore` 中自行配置。
