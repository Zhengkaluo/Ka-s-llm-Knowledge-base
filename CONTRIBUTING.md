# 贡献指南 · Contributing

感谢你对本知识库框架的兴趣！这是一个"协议为主、代码为辅"的项目，贡献方式和普通代码库略有不同。

## 这个项目的特别之处

本仓库的核心是一组**协议文件**（`SOUL.md` / `SKILL.md` / `AGENTS.md` / `PLATFORMS.md` / `_schema.md`）。改动它们等于改动"AI 的行为契约"，请格外谨慎。

## 可以贡献什么

- **新的笔记模板**（`_templates/`）
- **新的工具能力**（`tools/` 下新增一个自包含的 `.md` 能力说明 + 对应脚本）
- **新的讨论角色**（`AGENTS.md` 的角色池）
- **新平台的适配说明**（`PLATFORMS.md`，比如某个新的 AI 工具）
- **翻译 / 文档改进**
- **bug 修复**（脚本、init 工具等）

## 提交前请确认

1. **不要提交任何个人内容或凭证**。`topics/` 真实笔记、`_vault/` 材料本体、`.config/` 凭证都不应进入 PR。
2. **改协议文件时，说明动机**。在 PR 描述里讲清楚"为什么这样改更好"，最好附一个使用场景。
3. **保持权限规则不被削弱**。`SOUL.md` 的"读自由 / 写预览 / 删禁止"是这套框架的安全底线，新增能力不应绕过它。
4. **示例内容必须虚构**。任何放进 `example/` 的内容都不能引用真实受版权保护的材料。
5. **遵循 `_schema.md`**。新增笔记类示例需符合既有 frontmatter 与命名约定。

## 工作流程

1. Fork 本仓库并新建分支
2. 提交改动（commit message 建议用 `类型: 内容` 格式，见 `SOUL.md` 的 Git 章节）
3. 发起 Pull Request，在描述中说明改了什么、为什么

## 行为准则

友善、就事论事、尊重他人。这是一个帮助大家更好地思考与记录的工具，讨论时也请秉持同样的精神。
